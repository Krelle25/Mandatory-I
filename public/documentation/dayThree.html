Loop Methods in JavaScript

Instead of using traditional for loops, the focus was on using built-in array methods:

  • .map()
  • .filter()
  • .reduce()
  • .find()
  • .sort()
  • .indexOf()

Core Rules

  1. Use loop methods whenever possible.
  2. Only use for loops for "finger counting" (not for data transformations).
  3. Use .map() instead of .forEach() if you need the returned data.

Example: Using .map()


const numbers = [1, 2, 3, 4, 5];

const doubledNumbers = numbers.map(number => number * 2);

.map() creates a new array and maps 1:1 from the original array.

Example: Mapping Objects

Loop methods are also useful when working with arrays of objects. In class we practiced modifying object properties using .map(): 


const balloonAnimals = [
    { type: "Koala", difficulty: 5.0 },
    { type: "Dog", difficulty: 2.5 },
    { type: "Giraffe", difficulty: 1.5, isTall: true }
];

// Task: set all difficulty levels to 3.0 except Koala

const difficultyAdjustedBalloonAnimals = balloonAnimals.map((balloonAnimal) => {
    if (balloonAnimal.type !== "Koala") {
        balloonAnimal.difficulty = 3.0;
    }
    return balloonAnimal;
});

Here .map() loops through every element and returns a new array with the updated values.

Ternary Operator

We also used ternary operator to write conditional logic in a more compact way.


// condition ? if true : if false
            

const difficultyAdjustedBalloonAnimalsOneLiner = balloonAnimals.map((balloonAnimal) => ({
    difficulty: balloonAnimal.type !== "Koala" ? 3.0 : balloonAnimal.difficulty,
    ...balloonAnimal
}));

The spread operator (...) copies the existing object properties, while the ternary operator changes the difficulty based on the condition.

Arguments Passed to Loop Methods

Loop methods automatically receive multiple arguments:


numbers.map((element, index, originalArray) => {
    console.log(element, index, originalArray);
});
  • element → the current element
  • index → the position in the array
  • originalArray → the full array being iterated

Why Loop Methods Are Preferred

A major takeaway from the lecture was that modern JavaScript prefers functional loop methods instead of traditional loops.

Some of the main advantages are:

  • Brevity – fewer lines of code
  • Clarity – the method name describes the intention
  • Fewer indexing mistakes
  • Easier access to elements and indexes
  • More readable and maintainable code

Example: Traditional Loop vs Loop Method


const numbers = [6, 7, 8, 9];

for (let i = 0; i < numbers.length; i++) {
    console.log(numbers[i]);
}

The same result using a loop method:


numbers.forEach((number) => console.log(number));

Using .forEach() communicates that we are simply iterating over the list.

Avoiding Side Effects

Traditional loops often end up modifying external variables, which can create side effects. Using loop methods like .filter() and .map() helps avoid this and keeps the code cleaner.


let oddIndexSideEffect = [];
let index = 0;

for (number of numbers) {
    if (index % 2 === 1) {
        oddIndexSideEffect.push(Number(number));
    }
    index++;
}

The same logic using .filter():


const oddNumbers = numbers.filter((element, index) => index % 2 === 1);

This approach is shorter, easier to read, and avoids unnecessary mutation.

Avoid Mutating Arrays

Another key idea was avoiding mutation of the original array.


for (index in numbers) {
    numbers[index] = numbers[index] * 2;
}

This modifies the original array. Instead we can use .map() to create a new array:


const doubledList = numbers.map((number) => number * 2);

This keeps the original data unchanged and returns a new transformed array.

Anatomy of a URL

We broke down the structure of a URL into its compenents:

Example:

https://www.google.com/search?hl=en&q=cool%20search

URL Parts

  • Protocol → https://
  • Subdomain → www
  • Domain → google
  • Top-level domain → .com
  • Path → /search
  • Path variable → /search/1
  • Query string → ?hl=en&q=cool%20search

Status Codes in Express

Status codes were actively used to indicate the result of HTTP requests.

Example (404 - Not Found):


app.get('/movies/:id', (req, res) => {
    const providedMovieId = Number(req.params.id);
    const foundMovie = movies.find(movie => movie.id === providedMovieId);


    if (!foundMovie) {
        return res.status(404).send({
            errorMessage: `No movie found by ID: ${req.params.id}`
        });
    }


    res.send(foundMovie);
});

Status Code Categories

  • 2xx → Success (e.g. 200 OK)
  • 3xx → Redirection
  • 4xx → Client errors (e.g. 404 Not Found)
  • 5xx → Server errors

Using proper status codes makes the API more predictable and aligned with REST principles.
